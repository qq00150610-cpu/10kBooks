# 社交系统

## 模块概述

社交系统为用户提供围绕阅读的社交互动功能，包括书评评论、动态分享、即时通讯、关注关系等，构建读者与作者、读者与读者之间的连接。

## 2. 书评评论

### 2.1 评论结构

```json
{
  "id": "review_001",
  "bookId": "b_10001",
  "userId": "u_12345",
  "parentId": null,
  "rating": 5,
  "content": "这是一本非常精彩的科幻小说！",
  "images": ["https://..."],
  "likeCount": 128,
  "replyCount": 23,
  "isSpoiler": false,
  "isPinned": true,
  "status": 1,
  "createdAt": "2024-01-15T10:30:00Z"
}
```

### 2.2 评论层级

```
根评论 (书评)
  ├── 回复 1
  │    ├── 子回复 1.1
  │    └── 子回复 1.2
  ├── 回复 2
  └── 回复 3
```

### 2.3 评论功能

| 功能 | 说明 |
|------|------|
| 发布评论 | 对书籍进行评分和评论 |
| 回复评论 | 对评论进行回复 |
| 编辑评论 | 修改自己的评论 |
| 删除评论 | 删除自己的评论 |
| 点赞评论 | 赞同评论观点 |
| 举报评论 | 举报违规内容 |
| 精华评论 | 优质评论标记 |

### 2.4 评分系统

| 星级 | 说明 |
|------|------|
| ⭐ | 不推荐 |
| ⭐⭐ | 一般 |
| ⭐⭐⭐ | 还行 |
| ⭐⭐⭐⭐ | 推荐 |
| ⭐⭐⭐⭐⭐ | 强推 |

## 6. 隐私与权限

### 6.1 隐私设置

| 设置项 | 选项 | 说明 |
|--------|------|------|
| 动态可见性 | 公开/好友/仅自己 | 动态谁可以看 |
| 阅读记录 | 可见/隐藏 | 是否显示阅读历史 |
| 位置信息 | 开启/关闭 | 是否显示附近的人 |
| 黑名单 | 列表管理 | 屏蔽用户 |

### 6.2 权限控制

| 权限 | 说明 |
|------|------|
| 可被关注 | 是否允许被关注 |
| 可发私信 | 是否允许被私信 |
| 可看动态 | 非好友能否看动态 |
| 可评论 | 是否允许被评论 |

```
用户举报 ──▶ 选择类型 ──▶ 填写说明 ──▶ 上传证据 ──▶ 提交
                                    │
                                    ▼
                        后台审核 ──▶ 确认违规 ──▶ 处理
                                    │
                                    ▼
                              通知举报人 ──▶ 处理结果
```

## 8. API 接口

### 8.1 评论接口

| 接口 | 方法 | 说明 |
|------|------|------|
| `/v1/books/{bookId}/reviews` | GET | 书评列表 |
| `/v1/books/{bookId}/reviews` | POST | 发布书评 |
| `/v1/reviews/{reviewId}` | PUT | 编辑书评 |
| `/v1/reviews/{reviewId}` | DELETE | 删除书评 |
| `/v1/reviews/{reviewId}/like` | POST | 点赞书评 |
| `/v1/reviews/{reviewId}/report` | POST | 举报书评 |

### 8.2 动态接口

| 接口 | 方法 | 说明 |
|------|------|------|
| `/v1/feed` | GET | 动态列表 |
| `/v1/feed` | POST | 发布动态 |
| `/v1/feed/{feedId}` | DELETE | 删除动态 |
| `/v1/feed/{feedId}/like` | POST | 点赞动态 |
| `/v1/feed/{feedId}/comment` | POST | 评论动态 |

### 8.3 消息接口

| 接口 | 方法 | 说明 |
|------|------|------|
| `/v1/conversations` | GET | 会话列表 |
| `/v1/conversations/{convId}` | GET | 消息历史 |
| `/v1/messages` | POST | 发送消息 |
| `/v1/messages/read` | PUT | 标记已读 |

### 8.4 通知接口

| 接口 | 方法 | 说明 |
|------|------|------|
| `/v1/notifications` | GET | 通知列表 |
| `/v1/notifications/count` | GET | 未读数量 |
| `/v1/notifications/read` | PUT | 标记已读 |
| `/v1/notifications/settings` | GET/PUT | 通知设置 |

## 9. 性能优化

### 9.1 消息系统优化

| 策略 | 说明 |
|------|------|
| WebSocket 连接 | 实时消息推送 |
| 消息压缩 | 减少网络传输 |
| 分页加载 | 消息历史分页 |
| 本地缓存 | 减少重复请求 |
| 心跳检测 | 保持连接活跃 |

### 9.2 Feed 流优化

| 策略 | 说明 |
|------|------|
| 拉模式 | 用户主动拉取 |
| 推模式 | 写时推送 |
| 混合模式 | 分级推送 |
| 缓存 | Redis 缓存热门动态 |
| 异步处理 | 点赞等异步化 |
