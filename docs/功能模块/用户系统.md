# 用户系统

## 模块概述

用户系统是平台的基础核心模块，负责用户的注册登录、身份认证、个人信息管理、权限控制等功能。支持多端登录、数据同步、安全防护等能力。

---

## 1. 功能特性

### 1.1 核心功能

| 功能 | 说明 | 优先级 |
|------|------|--------|
| 手机号注册登录 | 手机号+验证码/密码登录 | P0 |
| 邮箱注册登录 | 邮箱+密码登录 | P0 |
| 第三方登录 | Google/Apple/Facebook 登录 | P1 |

| 账号安全 | 密码修改、安全设置、登录记录 | P0 |

## 2. 用户流程

### 2.1 注册登录流程

```
┌─────────────────────────────────────────────────────────────────────────┐
│                           注册登录流程                                   │
└─────────────────────────────────────────────────────────────────────────┘

  ┌─────────┐
  │  用户   │
  └────┬────┘
       │
       ├──┐ 选择登录方式
       │  │
       │  ├─▶ [手机号登录] ──▶ 输入手机号 ──▶ 发送验证码 ──▶ 验证 ──▶ 登录成功
       │  │
       │  ├─▶ [密码登录] ──▶ 输入手机号/邮箱 + 密码 ──▶ 验证 ──▶ 登录成功
       │  │
       │  ├─▶ [第三方登录] ──▶ 选择平台 ──▶ OAuth 授权 ──▶ 获取用户信息 ──▶ 登录成功
       │  │
       │  └─▶ [注册] ──▶ 选择方式 ──▶ 填写信息 ──▶ 验证 ──▶ 注册成功 ──▶ 登录
       │
       ▼
  ┌──────────────────────────────────────────────────────────────────┐
  │                         登录成功后的处理                          │  │
  │  1. 生成 Access Token (2小时有效)                                 │  │
  │  2. 生成 Refresh Token (30天有效)                                 │  │
  │  3. 返回用户信息给客户端                                          │  │
  │  4. 记录登录日志                                                  │  │
  └──────────────────────────────────────────────────────────────────┘
```

## 5. 安全设计

### 5.1 密码安全

- 密码使用 bcrypt 算法加密存储
- 密码强度要求：8-20位，包含大小写字母和数字
- 登录失败 5 次后，锁定 15 分钟
- 敏感操作需验证密码或验证码

### 5.2 Token 安全

- Access Token 有效期 2 小时
- Refresh Token 有效期 30 天
- Refresh Token 仅能使用一次
- Token 存储在 HttpOnly Cookie 中

### 5.3 防刷策略

| 策略 | 限制 |
|------|------|
| 发送验证码 | 同一手机号 1 分钟 1 次，24 小时 10 次 |
| 登录 | 同一 IP 1 小时 20 次 |
| 注册 | 同一 IP 1 天 10 个账号 |
